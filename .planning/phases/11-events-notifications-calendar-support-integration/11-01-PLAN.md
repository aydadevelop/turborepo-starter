<objective>
Converge booking status side effects onto the typed event/workflow boundary.

Purpose: `booking.updateStatus` currently calls `notificationsPusher` inline with empty recipients — a dead code path bypassing both the notification delivery pipeline and calendar sync subscribers. Phase 11 success criteria require that confirmed and cancelled booking events flow through `emitDomainEvent` so all registered pushers (calendar sync, notification delivery) receive them from a single typed boundary.

Output:
- `booking.updateStatus` emits `booking:confirmed` / `booking:cancelled` via `EventBus` from `@my-app/events`
- `registerNotificationEventPusher` updated to accept `db` for booking recipient resolution (no more empty recipients)
- Server startup registers the notification event pusher alongside the calendar sync already present
- Test coverage verifying the event bus convergence path
</objective>

2. Change `mapEventToNotificationInput` from sync to async and accept db:
   ```typescript
   const mapEventToNotificationInput = async (
     event: DomainEvent,
     db: Db
   ): Promise<EmitNotificationEventInput | null> => {
     switch (event.type) {
       case "booking:confirmed":
       case "booking:cancelled": {
         const { bookingId } = event.data as { bookingId: string };
         const [row] = await db
           .select({ customerUserId: booking.customerUserId, organizationId: booking.organizationId })
           .from(booking)
           .where(eq(booking.id, bookingId))
           .limit(1);
         if (!row?.customerUserId) return null; // skip if booking not found or no customer
         return {
           organizationId: event.organizationId,
           actorUserId: event.actorUserId,
           eventType: event.type,
           idempotencyKey: event.idempotencyKey,
           payload: {
             recipients: [
               {
                 userId: row.customerUserId,
                 channels: ["in_app"],
                 title: event.type === "booking:confirmed"
                   ? "Booking confirmed"
                   : "Booking cancelled",
                 severity: event.type === "booking:confirmed" ? "success" : "info",
               },
             ],
           },
         };
       }
       case "booking:created":
       case "payment:captured":
       case "payment:failed":
         return null; // recipients not resolved yet — skip silently
       default:
         return null;
     }
   };
   ```
3. Update signature of `registerNotificationEventPusher` to accept optional db (second param preserves backward compat with queue as first):
   ```typescript
   export const registerNotificationEventPusher = (
     queue?: NotificationQueueProducer,
     db?: Db
   ): void => {
     const resolvedDb = db ?? defaultDb;
     registerEventPusher(async (event, q) => {
       const input = await mapEventToNotificationInput(event, resolvedDb);
       if (!input) return;
       await notificationsPusher({
         input,
         queue: (q as NotificationQueueProducer | undefined) ?? queue,
       });
     });
   };
   ```
