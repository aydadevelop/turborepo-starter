  <action>
Create `apps/web/src/components/public/BookingRequestPanel.svelte` as a self-contained public booking-intake component that accepts at least `listingId` and `listingName` as props.

The component must manage these responsibilities locally:
- `$state` for `startsAt`, `endsAt`, `passengers`, `contactName`, `contactPhone`, `contactEmail`, `timezone`, `notes`, and `specialRequests`
- a quote query backed by `orpc.pricing.getQuote`
- a slot-check query backed by `orpc.availability.checkSlot`
- a protected booking mutation backed by `orpc.booking.create`
- auth gating via `authClient.useSession()`

Behavior requirements:
- quote + slot-check stay public and only run once the user has selected a complete candidate slot
- the component shows pending, unavailable, error, and quote-success states inline
- the booking mutation sends `listingId` plus booking-intake fields only; do not add `organizationId` or `publicationId` props or hidden fields
- if the user is not authenticated when they try to submit, redirect them to `/login?next=...` instead of attempting an anonymous booking
- on successful booking, show a success state with the returned booking id / total and invalidate `orpc.booking.key()`

Keep this as a self-contained component. Do not push booking mutation logic up into the page component.
