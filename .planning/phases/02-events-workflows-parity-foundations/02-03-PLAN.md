**The parity pattern (key design decision):**
A parity check compares two functions on the same inputs. The "legacy truth" (`legacyFn`) is inlined as a TypeScript function that returns known-good output. The "extracted implementation" (`extractedFn`) is the new code being validated.

```typescript
// packages/db/src/test/parity.ts

/**
 * Parity test harness for brownfield domain extraction.
 *
 * Usage (in any domain package's __tests__/parity.test.ts):
 *
 *   const parityTest = createParityTest({
 *     domain: "bookings",
 *     description: "getActiveBookingCount returns same value as legacy",
 *     inputs: [{ listingId: "abc" }, { listingId: "def" }],
 *     legacyFn: async (input) => legacyGetActiveBookingCount(input),
 *     extractedFn: async (input) => newGetActiveBookingCount(input),
 *   })
 *
 *   it("parity: bookings.getActiveBookingCount", parityTest)
 */

import { expect } from "vitest"

export interface ParityDeclaration<TInput, TOutput> {
  /** Human-readable domain name, e.g. "bookings" or "catalog.listings" */
  domain: string
  /** What behavior is being validated */
  description: string
  /** One or more input values to test. Each is run through both fns. */
  inputs: TInput[]
  /** The legacy truth-source function. Returns known-good output for each input. */
  legacyFn: (input: TInput) => TOutput | Promise<TOutput>
  /** The extracted implementation under test. Must match legacyFn output. */
  extractedFn: (input: TInput) => TOutput | Promise<TOutput>
  /** Optional: custom equality check. Defaults to deep equals. */
  equals?: (legacy: TOutput, extracted: TOutput) => boolean
}

export interface ParityResult<TOutput> {
  domain: string
  input: unknown
  legacy: TOutput
  extracted: TOutput
  pass: boolean
}

/**
 * Returns a Vitest test function that, when passed to `it(...)`, runs the
 * parity check for every input and asserts that extracted output deep-equals
 * the legacy output.
 */
export const createParityTest = <TInput, TOutput>(
  decl: ParityDeclaration<TInput, TOutput>
): (() => Promise<void>) => {
  return async () => {
    for (const input of decl.inputs) {
      const [legacy, extracted] = await Promise.all([
        decl.legacyFn(input),
        decl.extractedFn(input),
      ])

      const pass = decl.equals
        ? decl.equals(legacy, extracted)
        : JSON.stringify(legacy) === JSON.stringify(extracted)

      // Provide a rich failure message
      expect(pass, [
        `Parity failure — domain: ${decl.domain}`,
        `Description: ${decl.description}`,
        `Input: ${JSON.stringify(input)}`,
        `Legacy: ${JSON.stringify(legacy)}`,
        `Extracted: ${JSON.stringify(extracted)}`,
      ].join("\n")).toBe(true)
    }
  }
}
```

**packages/db/src/__tests__/parity.test.ts**:

The canary checks that the schema exports exactly the expected table set — no more, no less. The "legacy truth" is the hardcoded list of tables that existed at the Phase 1 snapshot (from 01-01 baseline migrations). The "extracted implementation" enumerates actual exported schema objects.

```typescript
import { describe, it } from "vitest"
import * as schema from "../schema"
import { createParityTest } from "../test"

// The known-good Phase 1 baseline table names (truth source).
// Update this list when adding new tables — the act of updating documents the delta.
const PHASE_1_BASELINE_TABLES = new Set([
  "user", "session", "account", "verification", "organization",
  "member", "invitation",
  "listing", "listingTypeConfig", "listingPricingProfile",
  "listingPublication", "listingStaffAssignment", "listingAvailabilityRule",
  "listingAvailabilityBlock", "cancellationPolicy", "booking",
  "bookingCancellationRequest", "organizationPaymentConfig",
  "paymentProviderConfig", "organizationSettings",
  "notificationEvent", "notificationPreference",
  "assistantChat", "assistantMessage",
  "userConsent",
  "affiliateReferral", "bookingAffiliateAttribution", "bookingAffiliatePayout",
  "todo",
  "workflowExecution", "workflowStepLog",  // Added in 02-02 (if 02-02 ran first)
])

// Extracted: enumerate Drizzle table names from the current schema.
// Each Drizzle table object has a `[Symbol.iterator]` or is identified as a table via [$drizzle].
// Check for table identity using the "getSQL" method (all drizzle tables have it)
// or by checking the object has a ._.name property.
const getExportedTableNames = (): Set<string> => {
  const tables = new Set<string>()
  for (const [exportName] of Object.entries(schema)) {
    // Skip enums and non-table exports
    // Drizzle pgTable objects have a Symbol/property we can check.
    // Simplest: check if the name matches camelCase table convention.
    // We compare against PHASE_1_BASELINE_TABLES to see if any are missing.
    tables.add(exportName)
  }
  return tables
}

describe("Schema parity canary", () => {
  it(
    "parity: all Phase 1 baseline tables exist in current schema exports",
    createParityTest({
      domain: "db.schema",
      description: "Phase 1 baseline tables are all still present in schema exports",
      inputs: [null],
      legacyFn: () => [...PHASE_1_BASELINE_TABLES].sort(),
      extractedFn: () => {
        const exported = getExportedTableNames()
        // Check that every baseline table name exists in current exports
        return [...PHASE_1_BASELINE_TABLES]
          .filter((name) => exported.has(name))
          .sort()
      },
      equals: (legacy, extracted) =>
        JSON.stringify(legacy) === JSON.stringify(extracted),
    })
  )

  it(
    "parity: createParityTest helper detects a deliberate mismatch",
    async () => {
      // Self-validation: ensure the harness catches failures
      // Use try/catch because expect() throws on failure
      const failingTest = createParityTest({
        domain: "test.self-check",
        description: "Returns different values — harness must catch this",
        inputs: [1],
        legacyFn: (_n: number) => "legacy-value",
        extractedFn: (_n: number) => "different-value",
      })

      // The failing test should throw an assertion error
      let threw = false
      try {
        await failingTest()
      } catch {
        threw = true
      }

      // If it didn't throw, the harness is broken
      if (!threw) {
        throw new Error("createParityTest failed to detect output mismatch — harness is broken")
      }
    }
  )
})
```

IMPORTANT: The canary test compares a subset of schema exports. If `workflowExecution`/`workflowStepLog` (from 02-02) haven't been added yet, remove them from PHASE_1_BASELINE_TABLES. The canary checks that the legacy set is a SUBSET of current exports, not an exact match — this way adding new tables doesn't break the canary.

Adjust the `equals` comparator accordingly: `extracted.length >= legacy.length && legacy.every(t => extracted.includes(t))`.

A parity test compares the extracted implementation of a domain function against the known-good
